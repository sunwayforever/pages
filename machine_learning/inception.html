1 Inception

inception 系列中最早的模型是 GoogleNet (2014/9), 提出了 inception 模块.

后来 Rethinking the Inception Architecture for Computer Vision (2015/12) 提出了 inception_v2 和inception_v3, 其中 v2 提出了 batchnorm, v3 提出了 factoried conv, asymmetric conv

1.1.1.1 Factoried Conv

A 是 factoried conv, 是指用两个 3x3 的 conv 代替 5x5 的 conv, receptive field 不 变, 但 flops 大约变为原来的 18/25 = 0.72 倍

1.1.2 Efficient Grid Size Reduction

图中的 D, E 为了做 downsample, 但并不是直接使用 pooling, 因为:

  1. 先 conv 后 pooling 时计算量太大
  2. 先 pooling 后 conv 时丢失的信息太多, 导致特征信息丢失 (representational bottleneck)

inception_v3 提出一个 `Efficient Grid Size Reduction` 的方法: conv 和 pooling 同 时做, 即一路做 pooling, 另一路做 stride 2 的 conv2d, 然后两者 concat.

1.1.3 Auxiliary Classifier

inception 模型会输出两个 software, 分别为 F, G, 其中 F 称为 auxiliary classifier. F, G 训练时使用相同的标签, F 人为的增加了训练的难度, 相当于一种 regularization 的手段.
