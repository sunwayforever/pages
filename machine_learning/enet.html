1 ENet

https://arxiv.org/pdf/1606.02147.pdf 2016/06

https://github.com/TimoSaemann/ENet

ENet 是一个注重推理速度的 semantic segmentation 模型

ENet 做为一个 semantic segmentation 模型, 大部分和 scnn 类似, 只是它针对网络的结构做了许多 trick

1.1 Network

假设网络的输入为 512x512.

ENet 的网络基本就是一个标准的 encoder-decoder 结构, 并且参考了 resnet (2015/12) 和 inception_v2 (2015/12) 的设计

1.1.1 initial

输入数据为 (3, 512, 512), 通过 initial 层后变为 (16, 256, 256), 这里没有直接做 pooling 而是参考了 inception_v2 的设计: 类似于 inception_v2 的 inception module

为了避免 pooling 导致的特征信息丢失 (representational bottleneck):

  1. 一种方法是先增大 feature 规模 (例如通过 1x1 conv2d, 或者类似 pillar 的作法) 获得冗余数据, 然后再做 pooling.
  2. 另一种方法是类似 inception_v2 的做法: 使用并行 pooling, 即一路做 pooling, 另一路做 stride 2 的 conv2d, 然后两者 concat, 这种做法和 FCN 的 concat 两个 pooling 的做法有些类似.

1.1.2 bottleneck

bottlenet 类似于 resnet 的 bottlenet 结构, 做了一点修改:

bottleneck 可以有不同的 type:

  1. downsampling

    bottleneck 第一个 1x1 conv2 变成 2x2, stride=2, 且左边分支加上一个 maxpooling, 用来做一个 2x 的 downsample, 这种结构和 initial 的并行 pooling 结构类似

  2. dilated

    bottleneck 中间的 conv 变成 dilated conv, 这个和 dilated net 类似

  3. asymmetric

    bottleneck 中间的 conv 变成 factorized conv, 例如 5x5 kernel 变 5x1 和 1x5 两个kernel, 在保持 receptive field 大小不变的基础上降低了计算量. 参考了 inception_v2

  4. upsampling

    和 downsampling 相反: bottleneck 中间的 conv 变成 deconv, 同时左边分支加一个 max_unpooling

1.2 Upsample

upsamle 主要有四种方法:

  1. interpolation
  2. deconv

  3. unpooling

    unpooling 是对原值重复或补零,以填充出更多数据

  4. max unpooling

    如果前面是通过 max pooling 做的 downsample, 则后面可以用 max unpooling 做 upsample, 它与普通 unpooling 的区别是它会利用之前做 max pooling 时的索引
