1. GCC Target Tunning

1.1. cost 相关

cost 是指 `the relative speed of rtx on the target machine`

gcc 在实施许多优化时需要考虑 target 的 cost, 例如 `x*12` 有可能会被 expand 成 `((x<<2)+x)<<2`, gcc 会比较两个 rtx 的 cost 决定是否应用这个优化.

通过 `-mtune` 可以指定一些 cost, 例如: 

static const struct riscv_tune_param rocket_tune_info = {
    /* SFMode          DFMode*/
    {COSTS_N_INSNS(4), COSTS_N_INSNS(5)},   /* fp_add */
    {COSTS_N_INSNS(4), COSTS_N_INSNS(5)},   /* fp_mul */
    {COSTS_N_INSNS(20), COSTS_N_INSNS(20)}, /* fp_div */
    /* SIMode          DIMode */
    {COSTS_N_INSNS(4), COSTS_N_INSNS(4)},   /* int_mul */
    {COSTS_N_INSNS(6), COSTS_N_INSNS(6)},   /* int_div */
    1,                                      /* issue_rate */
    3,                                      /* branch_cost */
    5,                                      /* memory_cost */
    true,                                   /* slow_unaligned_access */
};

  • fp_\{add,mul,div\}, int_\{mul,div\}

    arith cost

  • branch_cost

    branch_cost 决定了某些操作可不可以使用 branch 来实现， 例如 abs(x), 当 branch cost 较小时，会通过 branch 来实现， 当 branch cost 较大时， 会通过 shift/xor/sub 实现

    另外, gcc 的 pass_data_tree_ifcombine 也会考虑 branch_cost 的影响: 当 branch_cost 较大时, 会把 `if (a) {if (b) {}}` 优化成 `if (a&b)`

  • memory_cost

    访存的 cost, 例如: 

    riscv_rtx_costs:
    /*...*/
    case MEM:
        /* addressing cost, 例如 lui/auipc/got/ */
        if ((cost = riscv_address_insns(XEXP(x, 0), mode, true)) > 0) {
            /* memory cost */
            *total = COSTS_N_INSNS(cost + tune_param->memory_cost);
            return true;
        }
    /*...*/

1.3. scheduler 相关

mtune 中的 issue_rate 和 scheduler 相关

gcc 通过 md 定义了 scheduler 需要的流水线相关的信息, 例如:

  1. latency
  2. functional unit
  3. resource

例如： 

// gcc md:
(define_insn_reservation "sifive_7_fdiv_s" 27
  (and (eq_attr "tune" "sifive_7")
       (eq_attr "type" "fdiv,fsqrt")
       (eq_attr "mode" "SF"))
  "sifive_7_B,sifive_7_fpu*26")

// llvm td:
defm FSQRT_S : FPUnaryOp_r_frm_m<0b0101100, 0b00000, FFINX, "fsqrt.s">,
               Sched<[WriteFSqrt32, ReadFSqrt32]>;
def : WriteRes<WriteFSqrt32, [SiFive7PipeB, SiFive7FDiv]> {
               let Latency = 27; let ResourceCycles = [1, 26]; }

另外， llvm 的 mtune 中还包含额外的信息： 

def RocketModel : SchedMachineModel {
    let MicroOpBufferSize = 0; // Rocket is in-order.
    let IssueWidth = 1;        // 1 micro-op is dispatched per cycle.
    let LoadLatency = 3;
    let MispredictPenalty = 3;
    let CompleteModel = false;
}
